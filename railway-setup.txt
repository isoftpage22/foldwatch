Foldwatch on Railway — quick checklist
========================================

Architecture: MySQL (Railway) + Backend (Docker) + Frontend (Docker).
Repo root must stay the build context (do NOT set Root Directory to apps/*).

--- Config as code (in this repo) ---
• /railway.json — backend: builder DOCKERFILE, dockerfilePath docker/backend.Dockerfile,
  watchPatterns so only backend-related paths trigger deploys.
• /apps/frontend/railway.json — same for frontend + docker/frontend.Dockerfile.

The GitHub-connected BACKEND service uses the default config at repo root (railway.json).
Railpack / “No start command” is avoided because builder is explicitly DOCKERFILE.

For a FRONTEND service: Railway → that service → Settings → Config-as-code (or “Config file”)
and set the path to the absolute repo path:

  /apps/frontend/railway.json

If you skip that, the frontend service would still read root railway.json and build the API image.

(Root Dockerfile symlink is optional extra; railway.json is the source of truth for builder.)

--- Variables (must still be set in Railway UI — not committed) ---
Secrets and ${{ … }} references cannot live safely in git.

On the BACKEND app service (not MySQL), replace MySQL with your DB tile name if needed:

  DATABASE_HOST     = ${{MySQL.MYSQLHOST}}
  DATABASE_PORT     = ${{MySQL.MYSQLPORT}}
  DATABASE_NAME     = ${{MySQL.MYSQLDATABASE}}
  DATABASE_USER     = ${{MySQL.MYSQLUSER}}
  DATABASE_PASS     = ${{MySQL.MYSQLPASSWORD}}

Also add from apps/backend/.env: AI_PROVIDER, GEMINI_API_KEY, GEMINI_MODEL, …

Optional: RAILWAY_SHM_SIZE_BYTES = 268435456 (Playwright / Chromium)

On the FRONTEND service (after config path is set):

  INTERNAL_API_URL = http://${{foldwatch.RAILWAY_PRIVATE_DOMAIN}}:${{foldwatch.PORT}}

Replace foldwatch with your backend service’s exact canvas name. Fallback:

  INTERNAL_API_URL = https://${{foldwatch.RAILWAY_PUBLIC_DOMAIN}}

--- Flow ---
1) Add MySQL (Database → MySQL).
2) Backend: GitHub repo, root directory empty — uses /railway.json automatically.
3) Frontend: new service, same repo, root empty — set config file /apps/frontend/railway.json,
   then add INTERNAL_API_URL + Generate Domain.
4) Generate Domain on backend if you need a public API URL.

Notes
-----
- Backend listens on process.env.PORT (Railway sets this automatically).
- Playwright image is heavy; pick a plan with enough RAM/CPU.
- Never commit real API keys; set secrets only in Railway Variables.
